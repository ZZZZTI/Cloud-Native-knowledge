### 防火墙

```Shell
ufw enable/disable             # 启用（开机自启）/禁用
ufw status [verbose/numbered]  # 查看状态（默认策略、日志级别/带编号显示规则）
ufw reset                      # 重置（清除所有自定义规则，恢复初始禁用状态）

# 默认策略： 阻止/允许 外部发起连接/主动对外连接
ufw allow ssh -> ufw default allow/deny incoming/outgoing

# 管理访问规则
ufw allow/deny/limit [服务名] / [端口/协议] / [from IP(to any port 端口 proto tcp)]
#   放行/拒绝/限制速率    服务     服务端端口               特定IP访问特定端口

# 删除访问规则
ufw delete [allow 22/tcp] / 3

# 日志管理
ufw logging on/off              # 开启(low)/关闭日志
ufw logging low/medium/high     # 日志级别：记录被阻止的包/被阻止+允许的包/所有包

# 查看日志
tail -f /var/log/ufw.log       # 实时监控
grep "UFW BLOCK" /var/log/ufw.log | tail -20  # 查看最近被阻止的连接


## 6. 高级配置
# ---------- IPv6支持 ----------
# 编辑 /etc/default/ufw，确保 IPV6=yes（默认已开启）
vim /etc/default/ufw
# 修改后重启：ufw reload

# ---------- 应用配置文件（预定义规则集） ----------
ufw app list                   # 查看所有已注册应用
ufw app info OpenSSH           # 查看应用详细信息
ufw allow 'OpenSSH'            # 通过应用名放行（等效于 allow ssh）
ufw allow 'Apache Full'        # 放行Apache的80和443

# ---------- 模拟运行（预览效果，不实际应用） ----------
ufw --dry-run allow 8080/tcp

# ---------- 规则持久化 ----------
# UFW规则默认自动保存，无需额外操作


## 7. 安全加固组合示例
# 场景：Web服务器最小化开放端口
ufw default deny incoming
ufw default allow outgoing
ufw allow ssh                  # 管理端口
ufw allow http                 # 80
ufw allow https                # 443
ufw limit ssh                  # SSH增加限速
ufw enable


## 8. 排错与常用检查
# 检查UFW是否运行
ufw status | grep -i active

# 检查规则是否生效
ufw status verbose

# 检查端口是否开放（外部测试）
nc -zv 目标IP 端口
telnet 目标IP 端口
nmap -p 22,80,443 目标IP

# 查看防火墙日志排错
tail -50 /var/log/ufw.log
grep "DPT=80" /var/log/ufw.log | tail -10   # 过滤特定端口

# 临时放行所有（紧急排错，风险极高！）
ufw disable


## 9. 核心原则
1. 永远先放行SSH，再启用防火墙（否则自己被踢）
2. 默认拒绝入站，按需放行（白名单原则）
3. 生产环境开启SSH限速（limit）
4. 定期检查日志：/var/log/ufw.log
5. 修改规则后立即验证：ufw status
```

### SSH加固

```Shell
【二、SSH 安全（最重要的守门员）】

【配置文件 /etc/ssh/sshd_config（改完执行 systemctl restart sshd）】

# 1. 禁止 root 直接登录（必备！）
PermitRootLogin no

# 2. 禁止密码登录，只用密钥（最安全，但千万别把私钥丢了）
PasswordAuthentication no
PubkeyAuthentication yes

# 3. 改掉默认端口（防扫描器暴力破解）
Port 2222                    # 改成一个 10000-65535 之间的端口

# 4. 限制登录失败次数和超时
MaxAuthTries 3               # 最多试3次
ClientAliveInterval 300      # 5分钟无操作踢下线
ClientAliveCountMax 2

# 5. 限定允许登录的用户（白名单）
AllowUsers zhangsan lisi     # 只允许这俩用户 SSH

# 改完检查语法：
sshd -t                      # 返回无报错再重启，否则把自己锁外面


【密钥登录配置流程（运维基本功）】
# 1. 客户端生成密钥对
ssh-keygen -t rsa -b 4096    # 一路回车，生成 ~/.ssh/id_rsa（私钥）和 id_rsa.pub（公钥）

# 2. 把公钥传到服务器
ssh-copy-id -i ~/.ssh/id_rsa.pub user@服务器IP -p 2222

# 3. 或者手动追加（没 ssh-copy-id 时）
cat ~/.ssh/id_rsa.pub | ssh user@服务器IP -p 2222 "mkdir -p ~/.ssh && cat >> ~/.ssh/authorized_keys"

## 3. SSH安全加固（重中之重）
- 配置文件：/etc/ssh/sshd_config
  关键项：
  PermitRootLogin no                    # 禁止root直接登录
  PasswordAuthentication no             # 禁用密码登录（改用密钥）
  PubkeyAuthentication yes              # 开启密钥认证
  Port 2222                             # 更改默认22端口（防扫描）
  MaxAuthTries 3                        # 最大重试次数
  ClientAliveInterval 300               # 空闲超时自动断连
  AllowUsers user1 user2                # 白名单用户（仅允许特定用户）
  DenyUsers / DenyGroups                # 黑名单
- 密钥管理：
  ssh-keygen -t ed25519 -C "comment"    # 生成更安全的ed25519密钥
  ssh-copy-id user@host                 # 上传公钥
  私钥权限必须为600（chmod 600 ~/.ssh/id_ed25519）
- 防暴力破解：安装 fail2ban 或 denyhosts
  fail2ban 配置 /etc/fail2ban/jail.local 开启 sshd 防护
```

## 5. SELinux / AppArmor（强制访问控制）
- SELinux（CentOS/RHEL默认）：
  状态查看：getenforce ; sestatus
  临时切换：setenforce 0（宽容模式） / setenforce 1（强制模式）
  配置文件：/etc/selinux/config （永久修改）
  排错：ausearch -m avc -ts recent        # 查看被拒绝的操作
        sealert -a /var/log/audit/audit.log  # 生成修复建议
  修复：chcon -t httpd_sys_content_t /var/www/html/  # 修改上下文
       restorecon -Rv /var/www/html/          # 恢复默认上下文
       semanage fcontext -a -t httpd_sys_content_t "/web(/.*)?"  # 永久变更
- AppArmor（Ubuntu/Debian默认）：
  aa-status 查看状态
  aa-enforce /etc/apparmor.d/usr.sbin.nginx   # 强制模式
  aa-complain /etc/apparmor.d/usr.sbin.nginx  # 投诉模式（仅告警）

## 6. 安全审计与日志监控
- 系统日志：/var/log/messages、/var/log/secure（认证）、/var/log/maillog
- 关键日志监控命令：
  tail -f /var/log/secure                   # 实时监控登录行为
  journalctl -xe -p err                     # 查看错误级别日志
  journalctl -u sshd -f                    # 查看服务日志
- auditd（内核级审计）：
  systemctl start auditd
  配置 /etc/audit/rules.d/audit.rules
  示例规则：
  -w /etc/passwd -p wa -k identity        # 监控passwd写操作
  -w /etc/sudoers -p wa -k sudoers        # 监控sudoers变更
  -a always,exit -S unlink -S rmdir -k delete  # 监控删除行为
  查看审计结果：ausearch -k identity
- 文件完整性检查（AIDE）：
  aide --init ; mv /var/lib/aide/aide.db.new.gz /var/lib/aide/aide.db.gz
  定期对比：aide --check （可配合cron做巡检）

## 7. 进程与端口安全
- 查看监听端口及对应进程：
  netstat -tunlp  或 ss -tunlp
  lsof -i :80      # 查看谁占用了80端口
- 查杀可疑进程：
  ps auxf | grep -v root   # 查看非root运行进程（可疑）
  kill -9 PID
  pkill -f process_name
- 进程资源限制（防fork炸弹）：
  /etc/security/limits.conf
  user1 soft nproc 100      # 单用户最多100个进程
  user1 hard nproc 200
  全局限制：/etc/security/limits.d/ 下配置

## 8. 系统更新与漏洞修复
- 更新源配置：
  CentOS：/etc/yum.repos.d/ ; yum update / dnf update
  Ubuntu：/etc/apt/sources.list ; apt update && apt upgrade
- 安全补丁专用（重要）：

## 9. 内核安全参数调优（/etc/sysctl.conf）
- 网络层防攻击：
  net.ipv4.tcp_syncookies = 1          # 防SYN Flood
  net.ipv4.conf.all.rp_filter = 1      # 防IP欺骗
  net.ipv4.icmp_echo_ignore_broadcasts = 1
  net.ipv4.conf.all.accept_redirects = 0   # 禁止ICMP重定向
  net.ipv6.conf.all.accept_redirects = 0
- 内存/进程保护：
  kernel.randomize_va_space = 2        # 开启ASLR（地址空间布局随机化）
  kernel.dmesg_restrict = 1            # 普通用户禁止查看dmesg
  kernel.kptr_restrict = 1             # 限制/proc/kallsyms

## 10. Web中间件与数据库安全（泛）

## 13. 核心原则
- 最小权限原则（POLP）：用户、进程只给所需的最小权限
- 纵深防御：防火墙+SSH加固+SELinux+审计，层层防护
- 定期巡检：每周做一次安全自查（配合脚本自动化）
- 日志异地备份：将/var/log同步到远程存储，防篡改

【八、运维必踩的 4 个安全大坑（能把人关外面）】

坑1：改 SSH 端口/防火墙，没开双窗口保命
# 现象：改完防火墙或 SSH 配置，执行 systemctl restart sshd 后断开了，再也连不上
# 解决：改任何远程连接相关的配置前，开两个 SSH 窗口（A 改配置，B 保持连接），改完用 B 窗口测一下新端口/新配置能通，再关 A。

坑2：SSH 禁止密码登录，但公钥没放进去
# 现象：PasswordAuthentication no 改了，但 authorized_keys 里没你的公钥，直接 bye bye。
# 解决：改配置前先确认能密钥登录，或临时加一个本地有密码的备用账户。

坑3：sudo 语法写错了，所有 sudo 命令都报错
# visudo 会检查语法，但要是强行用 vim /etc/sudoers 改错了，所有 sudo 用户全废了
# 解决：只能用 visudo 改，它会锁文件和检查语法。

坑4：SELinux 开着，改了 Nginx 网站根目录，访问 403
# 现象：权限 (chmod 755) 和属主 (chown nginx) 都对，但网页就是打不开
# 解决：setenforce 0 瞬间好了，就是 SELinux 的目录上下文问题。
# 快速修复：chcon -R -t httpd_sys_content_t /新的网站目录/
