> 启动流程，服务管理，运行级别

------

### 启动流程

#### Step 1：BIOS / UEFI（硬件自检）

```Shell
1.硬件自检（POST）：检查内存、键盘、硬盘等是否正常
2.找启动盘：按 BIOS 里设置的启动顺序（硬盘/U盘/网络）找引导设备
3.读 MBR（老）或 GPT（新）分区表，找到引导程序（GRUB）
===========================================================
装系统 ->进 BIOS 改启动顺序（按 F2/Del/ESC 进）
GRUB没出来（黑屏/报错）->BIOS/UEFI 设置问题，检查启动顺序，或硬盘坏了
```

#### Step 2：GRUB（引导菜单，选内核）

```Shell
1.加载 /boot/grub/grub.cfg（或 /boot/efi/ 下的配置）
2.显示启动菜单（让你选内核版本，或进救援模式）
3.把选中的内核（vmlinuz）和 initramfs（临时文件系统）加载到内存
===========================================================
启动时按 e 键进入 GRUB 编辑模式（改内核启动参数，进单用户/救援模式）
启动时按 c 键进入 GRUB 命令行（手动引导，救急用）
常见内核启动参数：
single 或 1：进单用户模式（忘记 root 密码时用）
init=/bin/bash：直接进 bash，跳过所有服务（系统起不来时救急）
rd.break：进 dracut 救援模式（挂载根目录前停下，排错神器）

"vmlinuz not found"->内核文件丢失，用安装盘进救援模式重新安装内核
                   ->或GRUB 配置文件损坏，用 grub2-mkconfig 重新生成
```

#### Step 3：内核启动 + initramfs（加载驱动，挂载根目录）

```Shell
1.内核解压并初始化（CPU、内存、中断、调度器）
2.加载 initramfs（内存中的微型根文件系统，包含硬盘驱动和挂载工具）
3.在 initramfs 里加载硬盘驱动 → 挂载真正的根目录（/）
4.挂载成功后，切换根（switch_root）从 initramfs 切到真正的 /
===========================================================
"Starting dracut initqueue" ->磁盘没识别或 LVM 卷没激活
"Kernel panic - not syncing"->initramfs 损坏或根目录挂不上
                            ->进救援模式执行：update-initramfs -u
```

#### Step 4：启动 PID=1

```Shell
1.内核加载完根目录后，执行 /sbin/init（即 PID=1）。
2.新系统（systemd）：/lib/systemd/systemd（或 /usr/lib/systemd/systemd）
3.读取默认 target（systemctl get-default）
4.并行启动 target 下的所有依赖单元（多线程，快）
5.通过 socket、dbus、挂载点等触发按需启动（更灵活）
6.日志交给 journald 统一管理
===========================================================
启动卡住->看 PID=1 是谁：ps -p 1,journalctl
```

#### Step 5：启动所有服务 + 显示登录界面（getty）

```Shell
1.所有系统服务启动完成（网络、SSH、数据库、Web 等）
2.启动 getty（终端登录进程），显示 login: 提示符
3.如果是图形界面，启动 display manager（gdm/lightdm），显示登录界面
4.用户输入用户名密码 → 启动 Shell → 完成
===========================================================
"Failed to start xxx"-> 按Ctrl+Alt+Del重启,GRUB菜单按e,在内核末尾加
                        systemd.unit=emergency.target
                     -> 进救援模式后，systemctl disable 出问题的服务，或修复配置
                     
忘记 root 密码-> GRUB编辑内核行，末尾加 rd.break
   mount -o remount,rw /sysroot
   chroot /sysroot
   passwd root
   touch /.autorelabel
```

### 服务管理

```Shell
systemctl

# 查看状态
systemctl status nginx                     # 查看服务状态（是否运行、进程PID、最近日志）
systemctl is-active nginx                  # 只看是否运行（返回 active/inactive，脚本里用）
systemctl is-enabled nginx                 # 只看是否开机自启（返回 enabled/disabled）
systemctl list-units --type=service        # 列出所有正在运行的服务
systemctl list-units --all --type=service  # 列出所有服务（含停止的）

# 起停管理
systemctl start nginx              # 立即启动
systemctl stop/kill nginx          # 优雅/强制停止
systemctl restart nginx            # 重启
systemctl reload nginx             # 平滑重载配置（不中断服务，仅部分服务支持）
systemctl reload-or-restart nginx  # 能reload就reload，不能就restart
systemctl daemon-reload            # 重新加载所有服务配置文件（改了配置文件必执行）

# 开机自启
systemctl enable nginx             # 设置开机自启
systemctl disable nginx            # 取消开机自启
systemctl enable --now nginx       # 设置自启+立即启动（合体技，省一行命令）

# 屏蔽/解屏蔽
systemctl mask nginx               # 彻底锁死服务（任何人都不能启动，软链接到 /dev/null）
systemctl unmask nginx             # 解锁


journalctl

journalctl -u [服务] [选项]
-f                                 # 实时跟踪日志（类似 tail -f）
-n 50                              # 查看最近 50 行
--since "2026-06-20 10:00:00"      # 查指定时间之后的日志
--since "1 hour ago"               # 查最近1小时的日志
-p err                             # 只看错误级别以上的日志
```

### 切换运行级别

```Shell
# 级别
poweroff.target      # 关机
rescue.target        # 单用户救援模式
multi-user.target    # 多用户命令行模式（服务器默认）
graphical.target     # 图形界面模式
reboot.target        # 重启

# 切换
systemctl get-default           # 查看启动模式
systemctl set-default [级别]     # 设置为开机启动（服务器标配）
systemctl isolate [级别]         # 立即切换

# 查看启动耗时
systemd-analyze                  # 看整体启动时间
systemd-analyze blame            # 看每个服务启动耗时（排慢启动的神器！）
systemd-analyze critical-chain   # 看关键链，找出拖慢启动的元凶
```
