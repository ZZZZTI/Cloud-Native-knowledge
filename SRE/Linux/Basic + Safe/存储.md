> 物理磁盘 → 分区 → 文件系统 / 逻辑卷(LVM) / RAID → 挂载点 → 使用

------

### 磁盘

```Shell
# 查看磁盘信息
lsblk [-f]                # 查看所有磁盘和分区概览（带文件系统类型和UUID）
df [-Th] [挂载点]          # 查看磁盘空间使用情况（带文件系统类型）
fdisk -l /dev/sda         # 查看指定磁盘分区表
parted -l                 # 查看所有磁盘分区表（支持GPT）
smartctl -a /dev/sda      # 查看硬盘健康状态	
iostat -x 1               # 看磁盘使用率（%util接近100%说明磁盘忙不过来）
iotop                     # 看哪个进程在疯狂读写磁盘（交互式）

# 磁盘分区（分区工具：fdisk（交互式）、parted（支持脚本）、gdisk（GPT专用））
lsblk                     # 1. 确认新盘已识别
fdisk /dev/sdb            # 2. 进入分区工具
n      # 新建分区
p      # 查看分区表
d      # 删除分区
w      # 保存并退出
q      # 不保存退出
n → p → 回车 → 回车 → 回车 → w
partprobe /dev/sdb        # 3.内核重新读取分区表
lsblk                     # 4.查看结果
```

### 文件系统与 LVM（动态扩缩存储）

```Shell
# 查看信息
pvs/vgs/lvs []
pvdisplay/vgdisplay/lvdisplay                # 查看所有卷信息
df -i                                        # 查看inode使用
e2fsck  /dev/vg_name/mountpoint              # 检查文件系统


# 创建 LVM
pvcreate /dev/sdX1                           # 从分区创建 PV（物理卷）
pvmove /dev/sdX1                             # PV 更换磁盘
vgcreate vg_name /dev/sdX1                   # 创建 VG（卷组）
vgreduce vg_name /dev/sdX1                   # 移除 VG
lvcreate -L 10G -n lv_name vg_name           # 创建 LV（逻辑卷）
# 创建文件系统ext4/xfs
mkfs.ext4/xfs /dev/vg_name/lv_name   
ext4：fsck -t ext4 /dev/sdX1                 # 检查与修复（先umount）
XFS：xfs_repair /dev/sdX1
# 挂载使用
mount /dev/vg_name/lv_name  /mountpoint      # 挂载到/mountpoint


# LVM扩容
lvextend -L +5G/10G /dev/vg_name/lv_name     # 给 lv_name 增加 5G
# 文件系统扩展
resize2fs  /dev/vg_name/lv_name              # 扩容ext4格式
xfs_growfs /mountpoint                       # 扩容xfs格式
# 缩容（慎用）（xfs 不支持）
umount /var/www                              # 取消挂载
resize2fs /dev/vg_data/lv_www 10G            # 缩文件系统
lvreduce -L -5G /dev/vg_data/lv_www          # 再缩 LV


# 创建快照（-s 快照，-L 大小通常为原卷15%-20%，-n 名称）
lvcreate -L 5G -s -n lv_name_snap /dev/vg_name/lv_name
# umount->恢复快照->mount
lvconvert --merge /dev/vg_name/lv_name_snap
# 删除快照（释放空间）
lvremove /dev/vg_name/lv_name_snap
```

### 挂载使用

```Shell
mount                  # 查看已挂载的设备

# 临时挂载卸载（重启失效）
mount [选项] [磁盘路径] [挂载点]
mount -t ext4          # 指定文件系统类型
mount -o rw,noatime    # 挂载选项（rw=读写，noatime=不更新访问时间）
umount [挂载点/设备名]   # 卸载
umount -l /mnt/data    # 懒卸载

# 永久挂载（编辑/etc/fstab）
mount -a                                    # 测试：尝试挂载所有 fstab 条目，有报错赶紧改
blkid /dev/vg_data/lv_www                   # 查看 UUID
UUID=设备  挂载点  类型  选项  dump  fsck顺序   # 写入 /etc/fstab（一行）
mount -o remount,rw /                       # 根目录重新挂载为读写（改错）
```

### 网络存储

```Shell
NFS（网络文件系统）

编辑/etc/exports->/share  *(rw,sync,no_root_squash)   # 服务端配置
exportfs -a                                          # 生效
systemctl start nfs-server                           # 启动服务
mount -t nfs server:/share /local                    # 客户端挂载
```

### 磁盘列阵（软RAID）

```Shell
# 创建RAID5
mdadm -C /dev/md0 -l 5 -n 3 /dev/sdX1 /dev/sdY1 /dev/sdZ1
cat /proc/mdstat                    # 看软RAID状态        
mdadm --detail /dev/md0             # 看具体信息
mdadm --remove /dev/md0 /dev/sdX1   # 移除故障盘
mdadm --add /dev/md0 /dev/sdX2      # 添加新盘
mdadm -S/A /dev/md0                 # 停止/启动阵列
```

### 基础

```Shell
# 存储结构
分区 = 给硬盘切蛋糕（物理切块）
LVM卷 = 把切好的蛋糕块灵活合并/拆分（逻辑弹性）
挂载 = 给切好的蛋糕块开个门（连到目录上就能用）

物理磁盘：/dev/sda、/dev/sdb（SCSI/SATA盘），/dev/vda（虚拟化盘）
分区编号：/dev/sda1（第1个分区）、/dev/sda2（第2个分区）

# 分区表类型
- MBR：最多4个主分区（或3主+1扩展），不支持>2TB，用fdisk
- GPT：支持>2TB，最多128个分区，用gdisk 或 parted 

# 文件系统类型
  - ext4：通用，稳定，默认（云盘，系统盘）
  - XFS：大文件、高并发，性能优（服务器）
  - btrfs：支持快照、压缩、去重（较新）
  - vfat/FAT32、NTFS（需ntfs-3g）
  
# 文件系统结构
  - 超级块（Superblock）：文件系统元信息（备份：mke2fs -n 可查备份位置）
  - inode：存文件权限、大小、时间戳、数据块指针（不存文件名）
  - 块（Block）：通常4KB，文件系统读写最小单位
  - 目录项（dentry）：文件名→inode映射，缓存在内存
  
# LVM 三层结构
PV（物理卷）  = 原始分区（如 /dev/sdb1）打上 LVM 标记
VG（卷组）    = 多个 PV 组成的“存储池”
LV（逻辑卷）  = 从 VG 池子里切出来的“最终可用盘”，相当于虚拟分区

# 常见RAID级别：
RAID0（条带）：读写快，但一块盘坏全部数据丢（没冗余）
RAID1（镜像）：写慢读快，坏一块盘没事（1:1镜像，空间浪费一半）
RAID5（奇偶校验）：最少3块盘，坏一块没事，性能性价比高
RAID10（镜像+条带）：至少4块盘，又安全又快（有钱人的选择）
```
