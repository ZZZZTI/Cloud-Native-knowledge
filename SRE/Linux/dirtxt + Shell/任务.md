### vim（文本编辑）

```Shell
Insert:按 i/a/o 进入（写内容）
=========================================
Normal:按 Esc 进入（浏览、删除、复制、粘贴）
# 浏览
gg                    # 跳到文件第一行（看配置文件开头的全局变量）
G                     # 跳到文件最后一行（看日志末尾）
:123                  # 跳转到第123行（看报错行号时精准定位）
0                     # 跳到行首（光标所在行的第一个字符）
^                     # 跳到行首第一个非空字符（比0更常用）
$                     # 跳到行尾
Ctrl + d              # 向下翻半页
Ctrl + u              # 向上翻半页
# 搜索
/error                # 向下搜索 "error"，按 n 跳到下一个，按 N 跳到上一个
?error                # 向上搜索
# 操作
x                     # 删除字符
dd                    # 删除整行（剪切）
yy                    # 复制当前行
p                     # 粘贴
u                     # 撤销
Ctrl + r              # 撤销撤销
===========================================
Command在正常模式下按:进入（保存、退出、搜索替换）
# 退出
ZZ                    # 保存并退出
ZQ                    # 强制退出
:w (/tmp/backup.conf) # 保存(为备份文件)
# 设置
:set nu                # 显示行号（查报错行号，如 Nginx 配置报错）
:set nonu              # 取消行号
:set paste             # 进入粘贴模式（线上改 YAML 必须开！关闭自动缩进，防止格式错乱）
:set nopaste           # 退出粘贴模式（改完立马关）
:set fileencoding      # 查看当前文件编码（看是不是 UTF-8）
:set ff=unix           # 将换行符转为 Unix 格式（解决 Windows 传上来的 ^M 乱码）
:syntax on             # 开启语法高亮（看 JSON/YAML 结构是否错误）
:e ++enc=utf-8         # 用 UTF-8 编码重新加载文件（解决中文乱码）
# 替换
:%s/old/new/g          # 将全文的 old 替换为 new（不加 g 只换第一个）
:%s/old/new/gc         # 交互式替换（逐个确认，改核心配置强烈建议加 c）
:10,20s/old/new/g      # 仅替换第 10 行到第 20 行（精准打击）
:%s/^/# /g             # 全文行首加注释（#后面加空格，适合批量注释 Shell）
:%s/^# //g             # 批量去掉行首的 # 注释
:%s/\s\+$//g           # 删除所有行尾多余的空格（清理 YAML 和 Python 必备）
# 多窗口
:e /etc/hosts          # 在当前 Vim 中打开另一个文件（不退出当前编辑）
:bn                    # 切换到下一个文件（buffer next）
:bp                    # 切换到上一个文件
:ls                    # 列出当前所有打开的文件缓冲区
:sp filename           # 水平切分窗口（上下分屏，对比两个配置文件）
:vsp filename          # 垂直切分窗口（左右分屏）
Ctrl + ww              # 在分屏窗口间快速切换
:q                     # 关闭当前窗口
# 多列修改
# 操作流程（以给第10-20行加 # 注释为例）：
# 1. 按 Esc 确保正常模式
# 2. 输入 :10 回车（跳到第10行）
# 3. 按 Ctrl + v （进入列选择）
# 4. 按 10j   （向下移动10行，选中列）
# 5. 按 Shift + i （进入插入模式，光标在列首）
# 6. 输入 #    （写注释符）
# 7. 按 Esc   （等待1秒，所有选中的行首都会加上 #）
```

### crond（定时任务）

```Shell
# cron 服务管理
systemctl status crond      # 查看crond是否运行
systemctl start crond       # 启动crond
systemctl stop crond        # 停止crond
systemctl restart crond     # 重启crond
systemctl enable crond      # 设置开机自启

# 定时任务管理
crontab -e                  # 编辑当前用户的定时任务（/var/spool/crontabs/用户名）
crontab -l                  # 查看当前用户的所有定时任务
crontab -r                  # 删除当前用户的所有定时任务
crontab -u 用户名            # 操作指定用户的定时任务
grep CRON /var/log/syslog   # 查看cron任务日志

# 定时任务格式
[*  *  *  *  *]   [命令]/[脚本路径]   [>/>>]  [文件日志]  2>&1
 ┬  ┬  ┬  ┬  ┬
 │  │  │  │  └── 星期（0-7，0和7都代表周日）
 │  │  │  └───── 月份（1-12）
 │  │  └──────── 日期（1-31）
 │  └─────────── 小时（0-23）
 └────────────── 分钟（0-59）

时间
0 1 2             # 具体时间
*                 # 任意值（每分钟/每小时/每天...）
,                 # 枚举多个值，如 1,3,5
-                 # 范围，如 10-12
/                 # 步长，如 */5 表示每5个单位
@reboot           # 系统重启时执行一次
@daily/hourly..   # 每日/时...
```

### at（一次性任务）

```Shell
# at 服务管理
sudo apt install at -y
sudo systemctl start atd    # 启动 atd 守护进程
sudo systemctl enable atd   # 开机自启
sudo systemctl status atd   # 检查状态

# 管理 at 命令
at [时间] -f [有任务的文件]    # 创建一次性任务（交互式输入命令，Ctrl+D 结束）
at -l                       # 查看当前用户的所有待执行任务（等价于 at -l）
at -d <编号>                 # 删除指定编号的任务（等价于 at -d <编号>）
at -b                       # 系统负载低于 0.8 时执行任务（等价于 at -b）
at -c <编号>                 # 查看指定任务的详细内容
at -m                       # 任务执行后发送邮件（即使没有输出）
at -v                       # 显示任务将执行的精确时间（安排前）
at -q <队列名>               # 指定任务队列（a-z，字母越靠前优先级越高）
at -t [[CC]YY]MMDDhhmm      # 用固定格式指定时间（如 202607311430
echo "任务" ｜ batch         # 在系统平均负载低于 0.8 时才执行

# at 任务格式
> [/usr/bin/命令]/[脚本]  >>  [文件日志]
> ...
> ...
> <Ctrl+D>   # 按 Ctrl+D 保存并退出

# 时间格式
at 14:30/2:30pm       # 具体时分（24/12小时制）
at 10:00 07/31/26     # 具体日期：2026年7月31日上午10:00
at now/时分 + 2 minutes/hours/days/weeks/months/years # XX时间后的现在/时分
```
