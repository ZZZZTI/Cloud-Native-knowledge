> 2.看网卡端口，连通，路由，ssh，抓包

------

### 

```Shell
├─ 4. 网络管理与通信
│   ├─ 基础配置：ip, ifconfig, route, hostname
│   ├─ 连接查看：ss, netstat
│   ├─ 连通性测试：ping, traceroute, mtr
│   ├─ DNS解析：nslookup, dig, host
│   ├─ 端口测试：telnet, nc, curl
│   ├─ 抓包分析：tcpdump, wireshark (tshark)

# 网络

ip a                      # 查看 IP 地址
ping 8.8.8.8              # 测试外网
ping hadoop-namenode      # 测试内网服务器
nc -zv hostname port      # 测试端口是否开放
ss -tlnp | grep 8080      
netstat -tlnp | grep 8080 # 查看端口占用
nslookup baidu.com        
dig baidu.com             # DNS 解析
ip route                  # 查看路由（排查网络不通）
traceroute google.com     # 追踪路由，排查网络问题
curl -I http://api/health # 测试 HTTP 接口

# 检查端口是否开放（外部测试）

nc -zv 目标IP 端口
telnet 目标IP 端口
nmap -p 22,80,443 目标IP


【六、端口与网络连接排查（看有没有后门/异常连接）】

ss -tulnp                  # 查看所有监听的端口及对应进程（替代 netstat）⭐ 最常用
ss -tulnp | grep LISTEN    # 只看监听中的端口
ss -antp                   # 看所有 TCP 连接（含 Established）

lsof -i :80                # 看谁在用 80 端口
lsof -i :22                # 看谁在用 SSH 端口（发现暴力破解源IP）

netstat -anp | grep ESTABLISHED   # 老系统用（看建立的连接）

# 安全规则：服务器上不该开的端口（如 3306 对外、6379 对外）一律关掉。
```
