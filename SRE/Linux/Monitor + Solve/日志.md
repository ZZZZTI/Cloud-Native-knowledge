> 日志系统与轮替，常用日志

------

### 日志查询

```Shell
【一、Linux两大日志系统（新老并存）】

老派（rsyslog）              新派（systemd-journald）
----------------------------  ------------------------------
存文本文件（/var/log/xxx）    存二进制（journalctl 查看）
直接 cat / grep 就能看         必须用 journalctl 命令
轮替靠 logrotate             轮替靠 journald 自身的限额
代表文件：messages、secure   代表命令：journalctl -u

【现状】两个同时运行（默认都开着）。rsyslog 负责存文本，journald 负责存索引。


/var/log/syslog          # 系统通用日志（Debian/Ubuntu 系）
/var/log/secure          # 安全认证日志（SSH登录、sudo）⭐ 排查攻击必看
/var/log/auth.log        # 同 secure（Debian/Ubuntu）
/var/log/maillog         # 邮件日志
/var/log/cron            # 定时任务日志（看 crontab 有没有跑）
/var/log/nginx/          # Nginx日志（通常单独建目录）
/var/log/mysql/          # MySQL日志





【四、journalctl（systemd 日志查询）】

journalctl -u nginx -f           # 实时看 nginx 日志
journalctl -u nginx -n 100       # 看最近100行
journalctl -u nginx --since "1 hour ago"   # 最近1小时
journalctl -u nginx --since "2026-06-20 10:00:00"
journalctl -u nginx -p err        # 只看错误级别以上

# 查看整体日志占用空间
journalctl --disk-usage

# 限制 journald 日志大小（避免把 /var 撑爆）
# 修改 /etc/systemd/journald.conf ：
SystemMaxUse=500M           # 最多占 500M 磁盘
SystemMaxFileSize=100M      # 单个日志文件最大 100M

# 手动清理（不重启服务）
journalctl --vacuum-size=500M      # 清理到总大小不超过 500M
journalctl --vacuum-time=7d        # 清理 7 天前的日志


journalctl

journalctl -u [服务] [选项]
-f                                 # 实时跟踪日志（类似 tail -f）
-n 50                              # 查看最近 50 行
--since "2026-06-20 10:00:00"      # 查指定时间之后的日志
--since "1 hour ago"               # 查最近1小时的日志
-p err                             # 只看错误级别以上的日志







【五、运维必踩的 4 个日志大坑】

坑1：磁盘被日志撑爆（/var 100%）
# 现象：df -h 发现 /var 满了，服务报 "No space left on device"
# 原因：logrotate 没开，或 rotate 数量太大
# 解决：
du -sh /var/log/* | sort -hr | head -10   # 找哪个日志最大
# 紧急清理（把大日志先清空，不删文件）：
> /var/log/nginx/access.log                # 直接清空文件（前提是没被进程锁定）
# 或手动触发 logrotate：
logrotate -vf /etc/logrotate.d/nginx

坑2：logrotate 配置了 create，但程序没重载，日志写不进去了
# 现象：轮替后新日志文件是空的，旧日志还在写，但没内容了
# 原因：程序（如 Nginx/Java）一直拿着旧的文件句柄，轮替后创建了新文件，但程序还在往旧句柄里写（但旧文件已被移走了）
# 解决：
# - 方法1：用 copytruncate（复制+截断，不改变 inode，程序不用重启）
/var/log/nginx/*.log {
    copytruncate             # 替代 create，先复制再清空原文件
    daily
    rotate 7
    compress
}
# - 方法2：postrotate 里发送信号重载（如 kill -USR1 PID，或 systemctl reload nginx）

坑3：logrotate 的 postrotate 脚本报错，导致轮替失败
# 现象：日志长期不轮替，/var/log 越来越大
# 排查：
cat /var/log/messages | grep logrotate   # 看报错
# 调试：
logrotate -vf /etc/logrotate.d/xxx       # 看具体报错
# 常见错误：postrotate 里写的 nginx -s reopen 路径不对

坑4：journald 日志无限膨胀（/var/log/journal 几十G）
# 现象：du -sh /var/log/journal 发现巨大
# 原因：journald 默认无限制（RHEL 默认只用 10% 的 /var，但 UBUNTU 可能没有限制）
# 解决：
# 永久限大小：
vim /etc/systemd/journald.conf
SystemMaxUse=500M
SystemMaxFileSize=100M
# 重启生效：
systemctl restart systemd-journald
# 或立即清理：
journalctl --vacuum-size=500M


【六、排错三板斧（日志出问题照着查）】

1. 日志没生成 → 看服务是否启动，配置文件权限对不对
   systemctl status 服务名
   ls -l /var/log/xxx/          # 看属主和权限

2. 日志轮替没执行 → 手动触发看报错
   logrotate -vf /etc/logrotate.d/xxx
   cat /var/lib/logrotate/logrotate.status   # 看上次执行时间

3. 磁盘被日志撑爆 → 找大文件，清理或压缩
   find /var/log -type f -size +1G -exec du -h {} \;   # 找大于1G的日志文件

======================================================================
                      一句话总结（记这个就够了）
   文本日志在 /var/log/，系统日志用 journalctl -u 看。
   logrotate 控制日志轮替（/etc/logrotate.d/），核心参数 daily + rotate 7 + compress。
   程序不重载就写不进新日志？用 copytruncate 代替 create，或 postrotate 里重载服务。
   journald 限大小：改 /etc/systemd/journald.conf 的 SystemMaxUse。
======================================================================
```

### 日志轮转

```Shell
【配置文件位置】
/etc/logrotate.conf           # 主配置文件（全局默认）
/etc/logrotate.d/             # 每个服务单独配置（如 nginx、mysql）

【一个完整的 logrotate 配置例子（/etc/logrotate.d/nginx）】
/var/log/nginx/*.log {
    daily                     # 每天轮替一次
    rotate 7                  # 保留7个历史文件（保留7天）
    size 100M                 # 或者日志超过100M也触发轮替
    compress                  # 轮替后压缩（.gz）
    delaycompress             # 延迟压缩（保留最近一个不压，方便排查）
    missingok                 # 如果日志不存在，不报错
    notifempty                # 如果日志为空，不轮替
    create 640 nginx nginx    # 轮替后新建日志文件，指定权限和属主
    sharedscripts             # 多个日志文件轮替完，只执行一次 postrotate
    postrotate
        # 轮替完成后重启 nginx，让它重新打开日志文件句柄
        [ -f /var/run/nginx.pid ] && kill -USR1 `cat /var/run/nginx.pid`
    endscript
}

【核心参数速查表】
daily / weekly / monthly   # 轮替周期
rotate N                   # 保留 N 个历史文件
size 100M / 1G             # 超过大小立即轮替（与周期或的关系）
compress                   # 压缩成 .gz（省空间）
delaycompress              # 延迟一轮再压缩（配合 compress 用）
missingok                  # 文件不存在忽略（不报错）
notifempty                 # 空文件不轮替（节省I/O）
create mode user group     # 轮替后新建空文件，指定权限
sharedscripts              # 多个日志文件共用一个 postrotate 脚本（只执行一次）

【手动调试轮替（排错神器）】
logrotate -vf /etc/logrotate.d/nginx   # 强制轮替一次（-v 显示详情）
logrotate -d /etc/logrotate.d/nginx    # 调试模式（只看不动，dry-run）
```
