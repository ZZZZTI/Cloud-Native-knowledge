### 负载与整体

```Shell
# 查看系统平均负载->负载高看CPU和磁盘IO

uptime 
#时间       运行时间      用户数               1分钟   5分钟   15分钟
15:30:45   up 10 days, 1 user, load average: 2.5,    3.0,   2.8

w
# 用户     终端      来源IP      登陆时间   空闲时间   累计占用   当前占用   当前命令
  USER     TTY      FROM        LOGIN@   IDLE     JCPU      PCPU      WHAT
 parallel  pts/0  10.211.55.2   15:26    0.00s    2.75s     2.58s     sudo -i

# load average < 1 为正常，负载高的几种可能
CPU不够用（us/sy占比高） → 加CPU或优化程序
磁盘太慢（wa占比高） → 换SSD或减少磁盘读写
内存不够（频繁Swap） → 加内存

# 查看系统整体信息（内核版本、架构、发行版）
uname -a
cat /etc/os-release
```

### CPU

```Shell
# 实时 CPU 使用率

sar -u 2 5

top / htop [-u username]
# 第一行：负载
# 第二行：任务/进程数
# 第三行：%Cpu(s)
us = 应用程序占用（正常 < 70）
sy = 系统占用（正常 < 20，如果持续高说明有频繁系统调用）
id = 空闲（如果接近0说明CPU已满）
wa = I/O等待时间（如果 > 20 说明磁盘是瓶颈）
st = 被虚拟机偷走的时间（如果 > 10 说明母机超售严重）

us高 → 应用程序密集计算（死循环、复杂运算）→ 优化代码或扩容
sy高 → 系统调用频繁（上下文切换）→ 查看 vmstat 的 cs（context switch）列
wa高 → 磁盘慢 → 查 iostat，不是CPU的问题！
# 后面：
# 进程  用户          优先级         内存       状态  占用比           占用时间   命令名
  PID  USER      PR   NI   VIRT    RES    SHR  S   %CPU   %MEM     TIME+    COMMAND
  1234 root      20   0    1680m   123m   9.2m S   0.3    1.5      1:23.45  gnome-shell
  5678 parallel  20   0    12.4g   456m   8.3m S   0.0    5.6      0:56.78  chrome
# 状态：R（运行中/可运行）、S（可中断睡眠，通常空闲）、D（不可中断睡眠，通常在等硬盘）、Z（僵尸进程）、T（被暂停/跟踪）。
# P:按CPU使用率排序/M：按内存使用率排序/q：退出/按 c：显示完整的 COMMAND 命令路径


mpstat -P ALL [每秒输出] [总次数]   # 查看 CPU 使用率的快照
lscpu                             # 查看 CPU 的详细信息（型号、频率、核数）
sar -u -f /var/log/sysstat/sa10   # 查看某天的 CPU 数据
```

### 内存

```Shell
# 查看内存总量、已用、可用->内存高看Swap和缓存

sar -r 2 5

free -h
#              总内存       已使用       空闲      共享内存      缓存          可用内存
               total       used        free      shared      buff/cache   available
Mem:（物理内存） 7.6G        2.1G        1.2G      123M        4.3G         5.0G
Swap:（交换）   2.0G        0.0B        2.0G
# available < 10% → 内存紧张   Swap 使用持续增长 → 物理内存不够

vmstat 2
# 运行/阻塞队列     内存               交换分区读写    io读写     系统中断/切换   CPU 时间分配
 procs -----------memory----------   ---swap--  -----io----   -system--   ------cpu-----
 r  b   swpd   free   buff  cache     si   so    bi    bo     in   cs     us sy id wa st
 1  0    0     6200M  100M  500M      0    0     10    20     100  200    2  1  97  0  0


# 查看详细的内存信息（包括 swap、缓存）
cat /proc/meminfo
# 查看内存使用率最高的进程（带命令行）
top -o %MEM
# 查看共享内存段
ipcs -m
# 清理系统缓存（谨慎使用，应先 sync）
sync && echo 3 > /proc/sys/vm/drop_caches   # 清理 pagecache、dentries、inodes
```

### 进程

```Shell
# 查看进程
ps aux                 # 看所有进程 + 详细（最常用）
ps -ef                 # 标准格式，看 PID 和 PPID
pstree -p [1234]       # 树形显示所有进程及其PID

# 查看进程打开的文件
lsof [文件]             # 看谁在打开了什么文件
lsof -p 1234           # 看进程 1234 打开了哪些文件
lsof -i :80            # 看谁占用了 80 端口

# 进程的生命周期状态：STAT
R (Running)            # 正在运行或可运行
S (Sleeping)           # 可中断睡眠（等待事件，如等待用户输入）—— 最常见
D (Disk Sleep)         # 不可中断睡眠（等待磁盘IO，如数据同步）
Z (Zombie)             # 僵尸进程（已结束但父进程没收尸）
T (Stopped)            # 暂停（被 Ctrl+Z 挂起，或被信号暂停）

# 杀进程
kill  [1234/进程名]     # 正常终止
kill -9 1234           # 强制杀死
kill -1 1234           # 重载配置
kill -19 1234          # 暂停进程
kill -18 1234          # 恢复暂停进程
pkill -u zhangsan      # 杀掉某个用户的所有进程（排障时赶人用）
pkill -f "spark"       # 杀掉所有包含 spark 的进程

# 调整进程优先级（nice值：-20（最高优先级）到 19（最低优先级），默认是0）
nice -n [-20～19] [服务名]         # 启动时调整
renice -n [-20~19] [-p 1234]      # 运行时调整
```

### 磁盘I/O

```Shell
# 实时查看磁盘读写速度、I/O 队列、利用率->磁盘慢看IOPS和响应时间
sar -d 2 5

iostat -x [每秒输出] [总次数]

%util    = 磁盘繁忙程度 > 80% 说明磁盘忙不过来了
await    = IO响应时间 > 20ms 说明磁盘性能差（机械盘 < 10ms，SSD < 5ms）
r/s/w/s  = 每秒读写次数（IOPS）
rkB/s/wkB/s = 每秒读写数据量（吞吐量）

IOPS不够（小文件多） → 随机读写频繁，换SSD
吞吐量不够（大文件传输） → 换更高带宽的存储（如万兆NAS）
单个进程狂写（如日志写入太猛）→ iotop 找出来限制速度或异步写入


# 查看 I/O 排队情况（更详细）
iotop [-o]
# 查看磁盘的读写延迟
sar -d 1 5
# 查看磁盘的吞吐量
dd if=/dev/zero of=/tmp/test bs=1M count=1000 conv=fdatasync
# 查看 I/O 调度器
cat /sys/block/sda/queue/scheduler
# 查看详细的磁盘统计（包括读写次数、扇区数）
cat /proc/diskstats
```

### 磁盘空间

```Shell
# 磁盘
df -h                                  # 查看磁盘容量：Use%=100%，任何进程都无法写入文件
df -i                                  # 查看文件数量：重点关注根目录的 IUse%

# 目录
du -sh *                               # 查看当前目录下各文件/文件夹大小
du -sh /path/to/dir                    # 查看某个目录的大小（递归统计）
du -h /path/to/dir --max-depth=1       # 查看一级子目录大小
du -sh /data/* | sort -rh | head -10   # 找最大的10个目录
find / -type f -size +1G 2>/dev/null   # 找所有大于1G的文件

删了文件但 df 没释放空间
lsof | grep deleted > /proc/PID/fd/文件描述符
```
